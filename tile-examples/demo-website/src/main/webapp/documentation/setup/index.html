Installation and Compilation

The Aperture Tiles source code can be download from github. Aperture Tiles is made up of eight sub-projects:

  • math-utilities - contains some basic, underlying Java utilities to aid in processing data. Some angle utilities, some linear algebra, and a little statistics.
  • geometric-utilities - contains advanced math utitlities for processing geometry - mostly dealing with processing of geographic problems.
  • binning-utilities - the basic substrate of tiling, bin data structures, bin processing, and basic bin storage classes.
  • tile-generation - Spark-based tools to generate tile pyramids from raw data.
  • tile-service - a web service to serve tiles from tile pyramids to web clients.
  • tile-client - a simple web client to display tiles from tile pyramids.
  • tile-client-template - a starter template that makes it easier to create a tile-client and server application.
  • tile-examples - the Aperture Tiles documentation website and example applications.

All Aperture Tiles projects build with Apache Maven. We have tested building with Maven 3.1.0, but other versions may work. Ensure maven is installed and configured propertly on the system you are building Aperture Tiles. Prior to compiling Aperture Tiles we recommend determining the setup and configuration of the prerequisites required in the next section.

Prerequisites

This project has the following prerequisites:

  • Scala - version 2.9.3
  • Java - version of JDK 1.7+
  • Maven - version 3.1.0

Prior to compiling Aperture Tiles we recommend determining the setup and configuration of the following prerequisites. The components required will be dependent on the size of the data being processed:

  • Hadoop/HDFS (optional) - see relevent hadoop project for installation instructions. There are several flavors of Hadoop to choose from: Cloudera, Apache, MapR, HortonWorks, and probably a host of others. If you want to run large tiling jobs, you'll want a cluster of machines set up with Hadoop and Spark on which to run them. If you're running small jobs, or are willing to take a long time to do them, you can run Spark on a single node and just read to and write from the file system, in which case you don't need this step - but it makes things a lot easier, and lets you do a lot more.
  • HBase (optional) - Apache HBase acts much like a database which stores its data in Hadoop. If your tile sets are too many or too big to store on one machine, we recommend using HBase to store them.
  • Spark - Apache Spark is "... an open source cluster computing system that aims to make data analytics fast...". It also makes them very easy to develop and generalize. When you set up Spark, you have to configure the version of Hadoop with which it will be working. See Spark build instructions for how to do this. Aperture Tiles will work with versions of Spark 0.7.2 or greater. We recommend using version 0.8.1.
  • ApertureJS - To build the tile-service and tile-client projects Aperture must be installed in your maven repository. Aperture can be download from aperturejs.com. You will need to use the "Download Source" option, and run the maven installation (mvn install) to add the necessary jars to your maven respository.
  • Web Server - the Tile Server and client are built using the Restlet web framework and requires a servlet compatible web server. We recommend deploying to Apache Tomcat or Jetty.

Building the Project

Prior to building the project edit the <properties> sections of the aperture-tiles/pom.xml build file to specify the version of Hadoop and/or HBase one has installed. See comments in the file for valid settings for various versions. If you plan to run Spark in standalone mode and not utilize either, then it is safe to leave the default setting. For example to specify the Hadoop / HBase version is cloudera 4.4.0: 

<!-- cloudera version 4.4.0 -->
<hadoop-common-version>2.0.0-cdh4.4.0</hadoop-common-version>
<hadoop-core-version>2.0.0-mr1-cdh4.4.0</hadoop-core-version>
<hbase-version>0.94.6-cdh4.4.0</hbase-version>

To build all projects, execute 'mvn install' at a command prompt in the root project directory aperture-tiles. This will compile all the components and install .jar files for each project into your local maven repostitory on your build machine.

In actually running tiling jobs, the default run scripts in the tile-generation project expect to find the jars in your maven repository. Modifying them to look for the jars elsewhere if you don't want them there isn't hard, but you're on your own for that part. But if you want to go through all that, just use 'mvn package' instead.

See the tile generation or configuration documentation for specifics on generating tile sets from raw data and configuring a tile server and client to create a tile-based visual analytic application.
