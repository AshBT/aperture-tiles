Aperture Tiles Components

Aperture Tiles works by using three integrated components.

Distributed Tile Generation

The Aperture Tiles generation analytic smoothly processes live streams of data to support consistently fresh tiles. Cluster computing with Apache Spark is used to compute dynamic tiles from big data. The tile generation analytic seamlessly transforms live streamed raw data into AVRO data tile files with metadata descriptor files, summarizing all tile content. AVRO data files allow for language neutrality and support multiple summaries per bin allowing a flexible and complete analytic. Finished tiles are stored in HBase to allow efficient scalability.

Tile Server

The Aperture Tile server performs necessary tile preparation to ensure a fluid experience for the end user. The Aperture Tile Server defines a REST API for requesting tiles and rendering properties. The tile server works by retrieving AVRO data tile files and rendering appropriate PNG files for display in the tile client web app or external visualization applications. The Aperture Tile Server also has the ability to directly serve JSON tiles to easily integrate further processing.

Tile Client

The Aperture Tile client is a full featured interactive web app which requests tiles from the Aperture-Tile server and seamlessly renders them for the client. The web app is designed to promote exploratory data analysis by displaying high level patterns and summary statistics while preserving all fine detail at the appropriate zoom level.
